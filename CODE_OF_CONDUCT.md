# Code of Conduct

## Our Standard

Ograph is a technical project for graph rendering and interaction. Project
spaces should stay focused, respectful, and useful for contributors and users.

Expected behavior:

- be direct without being hostile,
- give technical feedback with enough context to act on it,
- accept maintainer decisions about package scope and release safety,
- keep discussions about people, identity, and private information out of
  public technical threads,
- report security-sensitive concerns privately instead of opening public issues.

Unacceptable behavior:

- harassment, threats, or personal attacks,
- sexualized language or imagery,
- repeated off-topic disruption after a maintainer asks for focus,
- publishing private information without explicit permission,
- abusing issue, pull request, or release channels to pressure maintainers.

## Enforcement

Maintainers may edit, hide, or remove comments; close issues or pull requests;
limit participation; or block contributors when behavior makes the project less
safe or less useful.

Because the repository is still pre-public, conduct reports should go to the
private maintainer channel that granted repository access. Before accepting
public outside contributions, maintainers should publish a dedicated private
conduct contact or organization-level reporting path.

Security-sensitive reports should follow `SECURITY.md`.
